# 详细流程参考

## 主流程

### Step 1：推送选题

1. 读配置，从**选题库**取数据
   - filter: `选题状态 is 待选`（⚠️ 用 `is`，不用 `isNot`）
   - 排序：今天采集 > 适合平台含小红书 > 推荐度降序；摘要为空或 `-` 降权
2. 取前 3 个，推送到群，末尾附选题库链接
3. 将 3 条状态更新为「已推送」
4. 写状态：`step=1, topics=[...], topicRecordIds=[...]`

### Step 2：选题分析

1. 将选中选题状态更新为「已使用」
2. 五维分析，生成 2 个方向：

```
选题：{title}  内容摘要：{summary}

| 维度 | 评分 | 依据 |
| 普遍痛点 | X/5 | 切中多少人的恐惧/焦虑 |
| 群体共鸣 | X/5 | 引发情绪共振+发声参与 |
| 身份认同 | X/5 | 命中哪个群体的标签 |
| 热点赋能 | X/5 | 能否借用当前热点 |
| 新知价值 | X/5 | 提供什么新东西 |

总分：X/25，潜力等级：S(20+)/A(15-19)/B(10-14)/C(5-9)/D(<5)
方向1：[角度]，适合[人群]，预计[效果]
方向2：[角度]，适合[人群]，预计[效果]
```

3. 写入**选题分析表**（字段映射见 tables.md）
4. 推送分析结果，末尾写「请回复 方向1 或 方向2」
5. 写状态：`step=2`，**等待用户回复**

### Step 3：生成文案

1. 生成小红书文案（标题 + 正文 + 标签）
2. 读 `references/sensitive-words.md`，执行敏感词替换
3. 推送文案到群
4. 写状态 → **立即执行 Step 4**

### Step 4：生成配图

1. 读 `references/brand.md` 获取风格和 prompt 模板
2. 用 `image_generate` 生成 4 张配图（aspectRatio: `3:4`），逐张生成，失败跳过
3. 上传流程：upload 到根目录 → 用 `feishu_drive_file` move 到目标文件夹
4. 推送配图结果；全部失败则推「配图生成失败，请手动补充」→ 继续 Step 5

### Step 5：写入内容生成库

1. 读 `references/tables.md` 获取字段格式
2. 写入内容生成库，发布状态填「待发布」
   - URL 字段格式：`{"link": "url", "text": "描述", "type": "url"}`
   - 正文中英文 `"` 替换为中文 `""`
3. 失败则跳过，继续 Step 6

### Step 6：推送草稿

```
📝 小红书完整草稿（可直接发布）

标题：xxx

正文：（完整正文，保留 emoji 和段落）

标签：#标签1 #标签2 #标签3

配图：
P1（描述）：飞书链接
...

📊 内容生成库：[查看表格](链接)
```

写状态：`active=false, step=0`。

## 仿写模式

1. 检查配置，不存在则暂存「仿写意图+URL」→ 走初始化向导 → 完成后自动继续
2. 用 `web_fetch` 抓取链接，提取标题、正文、博主信息、互动数据
3. 写入**选题库**（所有字段）
4. 自动五维分析，`selectedDirection = 「仿写：{原文标题}」`
5. 写入**选题分析表**
6. 写状态：`step=3, selectedTopic=原文标题, selectedDirection=...`
7. **直接跳 Step 3**，无停顿，不询问

---

## 数据回填模式

1. 在内容生成库按标题关键词搜索，展示当前数据
2. 请用户提供：发布时间、阅读量、点赞数、收藏数、评论数、分享数
3. 按爆款评级规则（见 tables.md）计算 S/A/B/C/D
4. 更新记录：发布状态→「已发布」、各项数据、爆款评级
5. 回复确认

---

## 复盘分析模式

1. 拉取「已发布」且有数据的记录（按时间范围过滤或取最近 30 条）
2. 分析维度：
   - **整体**：发布 N 条，S/A/B/C/D 分布，平均点赞/收藏
   - **爆款规律**（S+A）：选题方向/标题格式/发布时间共性
   - **踩雷规律**（C+D）：持续差的方向和失效的标题模式
3. 输出报告：

```
📊 小红书内容复盘报告
时间范围：XXXX-XX-XX ~ XXXX-XX-XX

📈 整体表现
发布：N 条 | 平均点赞：XXX | 平均收藏：XXX | 爆款率：X%

⭐ 爆款内容 TOP3
1、标题 - 点赞 XX / 收藏 XX
2、标题 - 点赞 XX / 收藏 XX
3、标题 - 点赞 XX / 收藏 XX

🔍 爆款规律
选题：XX方向被证明有效
标题：XX格式转化更好
时间：XX点-XX点发布表现最佳

⚠️ 踩雷总结
XX方向表现持续差，建议减少
XX标题模式失效，建议避开

💡 下一阶段建议
（基于数据给出 2-3 条具体建议）
```

4. 存入 `memory/xhs-review-{YYYY-MM}.md`

## 初始化向导

触发条件：收到触发词但配置文件不存在。

### 第一步：确认推送目标

```
👋 首次使用，需要初始化配置。

第 1 步：内容推送到哪里？
  A. 推送到某个飞书群（告诉我群名或群 ID）
  B. 在此私聊直接推送
```

- A + 群名 → `feishu_chat(search)` 搜索确认
- A + `oc_xxx` → `feishu_chat(get)` 验证
- B → `chat_mode=private`，`chat_id` 留空

### 第二步：确认多维表格

```
第 2 步：多维表格怎么处理？
  A. 帮我新建（推荐）
  B. 使用已有表格（提供 app_token）
```

- A → 执行 +init 建表
- B → 验证 app_token → 列出数据表 → 用户映射三张表 table_id → 验证字段完整性

### 第三步：确认配图文件夹（可选）

```
第 3 步：配图存哪里？
  A. 告诉我文件夹 token 或 URL
  B. 跳过（不保存到云空间）
```

### 写入配置

```json
{
  "app_token": "xxx",
  "tables": {"topics": "tblXXX", "analysis": "tblXXX", "content": "tblXXX"},
  "chat_id": "oc_xxx",
  "chat_mode": "group",
  "image_folder_token": "xxx",
  "bitable_url": "https://xxx.feishu.cn/base/xxx",
  "initialized_at": "ISO时间戳"
}
```

配置完成后自动开始正式流程（Step 1）。

## +init 建表

触发词：「初始化小红书表格」/ 「帮我建表」/ `+init`

1. `feishu_bitable_app(create, name="小红书内容管理")` → 获得 app_token
2. 读 `references/tables.md` 字段定义
3. 创建**选题库**（19 字段）：字段类型映射 Text→1, SingleSelect→3, MultiSelect→4, DateTime→5
4. 创建**选题分析**（9 字段）
5. 创建**小红书内容生成库**（含全部字段）
6. 删除/重命名默认的「数据表1」
7. 写入 `memory/xhs-pipeline-config.json`
8. 推送结果 + 继续初始化向导第一步
